# Плагин Tags Color Files

English | Русский

Данный плагин позволяет автоматически выделять файлы в вашем проводнике Obsidian разными цветами на основании имеющихся в них тегах. Цвета файлов определяются правилами, которые вы создаёте в настройках плагина, вводя тег и присваивая ему цвет. После добавления к файлу тега, для которого создано правило, этот файл окрашивается в соответствующий цвет в проводнике Obsidian.


## Фичи

1. Создание правил окрашивания файлов по принципу тег-цвет.

2. Приоретизация тегов, изменением их порядка в настройках перетаскиванием. Если в одном файле содержится сразу несколько тегов, для которых созданы правила, плагин будет выбирать приоритетный тег для окрашивания этого файла.

3. Выбор способа окрашивания: текста, фона, в виде цветных точек перед именем файла и в виде цветных точек после имени файла. При выборе вариантов перед или после текста, показывается до трёх цветных точек одновременно в том случае, если в одном файле присутствует насколько тегов, для которых созданы правила окрашивания.

4. Эскпорт и импорт настроек плагина в файл.


## Кейс использования плагина

Изначально, плагин был создан для моих личных нужд. При чтении литературы я помечаю некоторые цитаты тегами по типу `#ключевое` или `#не-согласен`, когда хочу выделить соответствующие мысли. Подобные цитаты мне удобно видеть в проводнике Obsidian, не прибегая к фильтрации по тегам через поиск. Именно эту проблему решает данный плагин.


## Известные проблемы и призыв к коллаборации

Я открыт к предложениям и новым идеям, расширяющим функционал плагина. Вот несколько идей:

1. Я не программист и код плагина практически полностью был сгенерирован нейросетью. Учитывая это, плагину требуется полный аудит кода для его чистки и оптимизации.

2. Способы окрашивания "Перед текстом" и "После текста" на данный момент достаточно плохо встроены в интерфейс проводника. Необходимо дополнить код так, чтобы эти способы окрашивания не конфликтовали с другими элементами интерфейса.

3. Необходимо разработать метод предотвращения создания двух правил для одного и того же тега.

4. Для улучшения пользовательского опыта, хотелось бы добавить подсказки при вводе тега при создании нового правила.
